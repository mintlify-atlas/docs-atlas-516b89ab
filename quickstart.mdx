This guide will help you get Zoo Arcadia running on your machine in just a few minutes. Choose between Docker (recommended) or local setup.

## Prerequisites

Before you begin, ensure you have the following installed:

- **PHP** 7.4+ (PHP 8.x recommended)
- **Composer** - PHP dependency manager
- **Node.js** and **npm** (includes npx)
- **MySQL/MariaDB** - Database server
- **Docker Desktop** (optional, for Docker setup)

## Quick Setup Options

<Tabs>
  <Tab title="Docker (Recommended)">
    Docker provides the fastest and most consistent setup experience.

    <Steps>
      <Step title="Clone the repository">
        ```bash
        git clone https://github.com/dumitrusf/zoo-Arcadia.git
        cd zoo-ARCADIA
        ```
      </Step>

      <Step title="Switch to Docker configuration">
        ```bash
        .\switch-to-docker.bat
        ```
        This script automatically updates your `.env` file with Docker database credentials:
        - `DB_HOST=db`
        - `DB_USER=zoo_user`
        - `DB_PASS=zoo_password`
      </Step>

      <Step title="Build and start containers">
        ```bash
        docker-compose up -d --build
        ```
        This command will:
        - Build the PHP/Apache container
        - Start MariaDB 11.4 database
        - Initialize the database with seed data
        - Compile assets automatically
      </Step>

      <Step title="Verify the installation">
        Check that all containers are running:
        ```bash
        docker-compose ps
        ```

        View logs if needed:
        ```bash
        docker-compose logs -f web
        ```
      </Step>

      <Step title="Access the application">
        Open your browser and navigate to:
        ```
        http://localhost:8080
        ```
      </Step>
    </Steps>

    ### Docker Management Commands

    <CodeGroup>
      ```bash Stop containers
      docker-compose down
      ```

      ```bash Rebuild after code changes
      docker-compose up -d --build
      ```

      ```bash Reset database (WARNING: deletes all data)
      docker-compose down -v && docker-compose up -d
      ```

      ```bash Apply specific SQL file
      docker exec -i zoo-arcadia-db mariadb -uzoo_user -pzoo_password zoo_arcadia < database/03_constraints.sql
      ```
    </CodeGroup>

  </Tab>

  <Tab title="Local Setup">
    Set up Zoo Arcadia directly on your local machine without Docker.

    <Steps>
      <Step title="Clone the repository">
        ```bash
        git clone https://github.com/dumitrusf/zoo-Arcadia.git
        cd zoo-ARCADIA
        ```
      </Step>

      <Step title="Install dependencies">
        Install PHP dependencies:
        ```bash
        composer install
        ```

        Install Node.js dependencies:
        ```bash
        npm install
        ```
      </Step>

      <Step title="Switch to local configuration">
        ```bash
        .\switch-to-local.bat
        ```
        This updates your `.env` file with local database settings:
        - `DB_HOST=localhost`
        - `DB_USER=root`
        - `DB_PASS=root`
        - `DB_NAME=zoo_arcadia`
      </Step>

      <Step title="Deploy the database">
        ```bash
        .\deploy_database.bat
        ```
        <Warning>
          This command recreates the database from scratch, removing any existing data.
        </Warning>
      </Step>

      <Step title="Compile assets">
        Build CSS and JavaScript files:
        ```bash
        npx gulp buildCss && npx gulp buildJs
        ```
      </Step>

      <Step title="Start the development server">
        ```bash
        php -S localhost:3001 -t public public/index.php
        ```
      </Step>

      <Step title="Access the application">
        Open your browser and navigate to:
        ```
        http://localhost:3001
        ```
      </Step>
    </Steps>

  </Tab>
</Tabs>

## Database Configuration

Zoo Arcadia uses environment variables for database configuration. The `.env` file is automatically managed by the switch scripts.

<CodeGroup>
  ```ini Docker Configuration
  DB_HOST=db
  DB_NAME=zoo_arcadia
  DB_USER=zoo_user
  DB_PASS=zoo_password
  ```

  ```ini Local Configuration
  DB_HOST=localhost
  DB_NAME=zoo_arcadia
  DB_USER=root
  DB_PASS=root
  ```
</CodeGroup>

## Default Credentials

<Note>
  Default user credentials are created during database seeding. Check the `database/06_seed_data.sql` file for the complete list of seeded users and their roles (Admin, Veterinary, Employee, Accountant).
</Note>

The application includes the following role types:
- **Admin** - Full system access
- **Veterinary** - Animal health and reports management
- **Employee** - Limited actions (maintenance, feeding)
- **Accountant** - Financial management

## Asset Compilation

Zoo Arcadia uses Gulp to compile SCSS and JavaScript files.

<CodeGroup>
  ```bash Production Build
  npx gulp buildCss && npx gulp buildJs
  ```

  ```bash Development with Watch
  npx gulp
  ```
</CodeGroup>

The development mode includes:
- File watching for automatic recompilation
- BrowserSync for live reload
- Source maps for debugging

## Common Issues

<AccordionGroup>
  <Accordion title="Port already in use">
    If port 8080 (Docker) or 3001 (local) is already in use, you can:
    
    - Stop the conflicting service
    - Change the port in `docker-compose.yml` (Docker) or use a different port with `php -S localhost:PORT`
  </Accordion>

  <Accordion title="Composer SSL certificate error (Windows)">
    See the [Installation Guide](/installation#ssl-certificate-error) for detailed troubleshooting steps.
  </Accordion>

  <Accordion title="Database connection failed">
    Verify your `.env` file settings match your environment:
    - For Docker: ensure containers are running with `docker-compose ps`
    - For Local: ensure MySQL/MariaDB service is running
  </Accordion>
</AccordionGroup>
